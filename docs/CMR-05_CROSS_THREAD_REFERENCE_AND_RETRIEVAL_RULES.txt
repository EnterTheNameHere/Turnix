===============================================================================
CORE META-RULES CMR-05 - CROSS-THREAD REFERENCE AND RETRIEVAL RULES
===============================================================================

Status:
-------
FROZEN

Description:
------------
Defines how authoritative Turnix documents are referenced,
retrieved, and continued across chat threads.

Purpose:
--------
This document defines how Turnix specification documents behave when they
are referenced, resumed, or extended across multiple conversation threads.

It defines:
- continuity of document identity and authority across threads
- reference by DOC_ID
- rules for partial quotation and excerpts
- continuation and resumption behavior
- prohibited assumptions about missing visibility
- clarification behavior for uncertain references
- handling of missing references and unknown entities

It does NOT define:
- document content or semantics
- formatting rules (see CMR-01)
- document title and preamble rules (see CMR-02)
- document creation and change discipline (see CMR-03)
- indexing rules and conflict resolution (see CMR-04)

Upstream Documents:
-------------------
- CMR-01
- CMR-02
- CMR-03
- CMR-04


===============================================================================
01. NORMATIVE LANGUAGE
===============================================================================

01.1. Turnix specifications express normative meaning through explicit,
      deterministic prose rather than through RFC-style keywords.

01.2. Normative meaning is expressed through clear actor-action statements
      that describe behavior, guarantees, constraints, or required
      handling.

01.3. Normative rules remain understandable without relying on special
      vocabulary markers.

01.4. Statements describing permitted alternatives express implementation
      freedom only within the constraints explicitly defined by the
      specification.

01.5. RFC-style requirement keywords carry no special meaning in Turnix
      specification documents.

01.6. The tokens "SHOULD" and "SHOULD NOT" are forbidden in authoritative
      documents because they introduce ambiguous advisory semantics.

01.7. When such tokens appear in draft text, the drafting author removes
      or rewrites them before the document is treated as authoritative.

01.8. Normative meaning derives from sentence structure and document
      context, not from reserved keywords.


===============================================================================
02. DOCUMENT CONTINUITY ACROSS THREADS
===============================================================================

02.1. Thread boundaries do not affect document identity or authority.

02.2. A specification document remains in force exactly as written
      regardless of:
      - which thread it was introduced in
      - whether it is reprinted in full
      - whether it is referenced indirectly

02.3. Absence of a document in the current thread does not imply absence
      of authority.

02.4. A document does not lose force because the current thread contains
      only excerpts, summaries, or references to it.

02.5. Cross-thread continuation preserves the same document unless an
      explicit accepted revision changes it under CMR-03 and CMR-04.


===============================================================================
03. REFERENCE BY IDENTIFIER
===============================================================================

03.1. Specification documents are referenceable by their DOC_ID values as
      defined in the Canonical Index (CMR-04).

03.1.1. Examples:
        - CMR-05
        - DA-09
        - ODA-E
        - IS-01

03.2. A reference by identifier implicitly refers to:
      - the document located at the Canonical Index file path for that
        DOC_ID
      - the full document content

03.3. Referencing a document by identifier does not reopen it for change.

03.4. LOCKED content remains LOCKED unless explicitly revised through the
      change process defined in CMR-03.


===============================================================================
04. PARTIAL QUOTATION AND EXCERPTS
===============================================================================

04.1. Quoting or printing part of a document does not supersede the rest
      of the document.

04.2. Partial excerpts are not treated as complete definitions.

04.3. If only a section is visible in the current thread, unseen sections
      remain authoritative and in force.

04.4. A quotation, summary, or excerpt does not narrow document scope
      unless the document itself explicitly does so.


===============================================================================
05. CONTINUATION AND RESUMPTION
===============================================================================

05.1. Work on an authoritative document may continue in a new thread.

05.2. Continuation identifies the DOC_ID.

05.2.1. Continuation states whether the work is:
        - review
        - reformatting
        - clarification
        - OPEN resolution
        - explicit revision

05.3. Silence about intent is treated as NO CHANGE.

05.4. Continuing discussion about a document does not by itself amend the
      document.

05.5. A document changes only through explicit revision handled under
      CMR-03 and reflected through CMR-04.


===============================================================================
06. PROHIBITED ASSUMPTIONS
===============================================================================

06.1. A language model assistant does not assume that:
      - a document is obsolete because it is not visible
      - conversational tone overrides written specification
      - newer chat messages implicitly amend documents

06.2. Authority flows only from:
      - the document content
      - the Canonical Index (CMR-04)
      - explicit revision instructions handled under CMR-03

06.3. A missing quotation, missing excerpt, or missing retrieval in the
      current thread does not create permission to improvise authoritative
      content.


===============================================================================
07. ERROR HANDLING AND CLARIFICATION
===============================================================================

07.1. When uncertainty exists about which Canonical Index entry applies,
      clarification is requested.

07.2. Guessing, reconstructing, or approximating missing authoritative
      text is explicitly forbidden.

07.3. When the current thread does not provide enough information to
      identify the intended authoritative document, the assistant does
      not substitute another document, rule, or concept.

07.4. Clarification resolves identifier uncertainty. Clarification does not
      create new authority.


===============================================================================
08. MISSING REFERENCES AND UNKNOWN ENTITIES
===============================================================================

08.1. If a term, identifier, or concept appears to reference an
      authoritative document, rule, or entity, but no corresponding entry
      exists in the Canonical Index (CMR-04), the reference is treated as
      missing.

08.2. Missing references are not invented, inferred, or approximated.

08.3. In the presence of a missing reference, the assistant does exactly
      one of the following:

      A) Ask the user to identify or confirm the intended document or
         entity.

      B) Inform the user that the reference is missing and cannot be used
         until it is identified or added to the Canonical Index.

08.4. The assistant does not:
      - invent, infer, or approximate missing references
      - assume default behavior
      - reuse a similar existing document
      - silently substitute a nearby concept

08.5. This rule applies only to:
      - DOC_ID values
      - authority claims
      - cross-document semantic dependencies

08.5.1. A cross-document semantic dependency exists when a document claims
        that its meaning relies on a rule, definition, or constraint in
        another document identified by DOC_ID.

08.6. This rule does not apply to:
      - formatting tasks
      - pure rewording
      - explicitly exploratory OPEN sections


===============================================================================
09. RETRIEVAL OUTCOME AND AUTHORITY PRESERVATION
===============================================================================

09.1. Retrieval failure in the current thread does not invalidate an
      authoritative document.

09.2. Failure to print or quote a document in full does not weaken the
      authority of any unseen portion.

09.3. A successful retrieval does not change document meaning. It only
      exposes already authoritative text.

09.4. Retrieval behavior preserves authority boundaries rather than
      creating new ones.
