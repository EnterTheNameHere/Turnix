Runtime design 2 open question map
===================================

Purpose
-------
This file captures open questions identified during the runtime design 2
conversation pass.

This file is a planning artifact only. It is not an authoritative Turnix
specification document and does not override indexed CMR, DD, DA, ODA, or IS
documents.

Loaded context used for this pass
---------------------------------
- AI_WORKFLOW_BOOTSTRAP
- CMR-04
- DA-20 AppInstance Lifecycle & Runtime Identity
- DA-21 RPC, Channels, Transports & Message Flow
- DA-22 UI, Views & Frontend Runtime
- DA-23 Execution Boundaries, Adapters and Code Runtimes
- DA-27 Adapter Lifecycle, Trust and Traceability

Primary gap categories
----------------------

Immediate recommended next step
-------------------------------
Create a narrow ownership plan before drafting new authoritative semantics.

The first promoted document is likely a Runtime Host / Workspace Ownership
design area because multiple existing open questions point there, and DA-27
currently defines runtime-host meaning only where needed to preserve adapter
authority boundaries.
