===============================================================================
DESIGN AREA TRACE-04 - TRACE REASONS AND ATTRIBUTES
===============================================================================

Status:
-------
EVOLVING

Description:
------------
Defines trace reason field semantics and trace attribute field semantics.

Purpose:
--------
This document defines the reason and attribute mechanics formerly contained in
DA-03 sections 09 through 10.

It defines:
- the trace reason field
- reason ownership boundaries
- trace attributes
- attribute ownership boundaries
- linkability through structured attributes

It does NOT define:
- a global catalog of domain-specific reason values
- object-specific reason naming schemes
- every object-specific attribute set
- concrete serialization formats

Upstream Documents:
-------------------
- TRACE-00
- TRACE-01
- TRACE-02


===============================================================================
01. TRACE REASON FIELD
===============================================================================

01.1. reason is the machine-readable reason field on a trace event.

01.2. Reason values identify why an event was emitted.

01.3. Reason values are interpreted within the domain that owns them.

01.4. TRACE does not define a global catalog of domain-specific reason
      values.

01.5. TRACE does not define object-specific reason naming schemes.

01.6. TRACE does not require a domain to use a particular reason value for
      an object-specific transition.

01.7. The document that owns a domain, lifecycle, or workflow defines the
      reason values needed by that subject.

01.8. Subject-owned reason values are interpreted through the trace event
      mechanics defined by TRACE.

01.9. Reason examples in TRACE documents are illustrative when present
      unless the surrounding text defines them as TRACE-owned substrate
      reasons.

01.10. Illustrative reason examples do not create required domain coverage.


===============================================================================
02. TRACE ATTRIBUTES
===============================================================================

02.1. attributes is the structured key-value data attached to a trace event.

02.2. Attributes carry event-specific evidence.

02.3. Attributes support offline linking between trace evidence and Turnix
      objects.

02.4. Stable identifiers may appear in attributes when the emitting domain
      owns or knows those identifiers.

02.5. Identifier presence varies by domain.

02.6. TRACE does not define every object-specific attribute set.

02.7. The subject-owning document defines which attributes are required for
      its own events when it defines trace obligations.

02.8. Subject-specific attributes do not redefine the attribute field
      mechanics defined by TRACE.

02.9. Boundary events and mutation events remain linkable to their owning
      objects when the owning domain requires that linkage.


===============================================================================
03. LOCKED SUMMARY
===============================================================================

03.1. LOCKED:
      - reason is a machine-readable event field
      - reason values identify why an event was emitted
      - domain documents own their own reason values
      - attributes carry event-specific structured evidence
      - domain documents own their required object-specific attributes
      - subject-specific attributes do not redefine TRACE attribute mechanics
