Turnix cross-document consistency sweep helper
==============================================

Purpose
-------
This helper defines a reusable consistency sweep for batches of related
Turnix document changes.

Use this helper when:
- several related documents changed in one pass
- one document became more precise and neighboring documents may lag behind
- authority boundaries may have shifted across a document set
- a local change may have caused spillover consistency issues elsewhere

Core principle
--------------
Local correctness is not enough.
After a related document batch, verify that the set still behaves as one
coherent specification system.

Consistency sweep checks
------------------------
Check the following across the touched document set:

1. Authority-home alignment
- Does each concept still have one clear authority home?
- Did one document silently take ownership from another?
- Are neighboring documents now duplicating meaning inconsistently?

2. Terminology alignment
- Are the same concepts named consistently?
- Did one document become more concrete while another stayed vague in a
  conflicting way?
- Did broader terms appear in one place but not another?

3. Dependency alignment
- Do cross-references still match current ownership?
- Did a document become [constrained-by] another without the neighboring
  docs reflecting that change?
- Did a newer document become more precise than its upstream/downstream pair
  in a conflicting way?

4. Guarantee alignment
- Do the touched documents still express the same guarantees where they
  overlap?
- Did one document become stricter or looser in a way that now conflicts with
  another?

5. Spillover detection
- Which untargeted documents may now need a small alignment pass?
- Is the inconsistency real enough to fix now rather than defer?

What to report
--------------
When reporting sweep results, distinguish:
- documents that are consistent
- minor wording-only differences that do not change semantics
- real consistency failures
- likely spillover docs that should be checked next

Do not call the set consistent merely because no direct contradiction was
obvious at first glance.
Check precision mismatches and ownership drift too.

Working mantra
--------------
A document batch is finished only when the documents still agree on who owns
meaning, how terms are used, and where decisions come from.

End
---
This helper defines reusable consistency-sweep workflow only.
It does not define Turnix runtime semantics.
