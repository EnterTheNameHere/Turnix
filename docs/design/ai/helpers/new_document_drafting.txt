Turnix new document drafting helper
===================================

Purpose
-------
This helper defines how to draft a new Turnix document when no prior document
already owns the target meaning.

Use this helper when:
- creating a new DA, DD, ODA, IS, or helper document
- splitting one overloaded document into several new documents
- drafting a document for a newly locked concept with no existing home

Core principle
--------------
Create the smallest correct authority home for the concept.
Do not silently import neighboring meaning unless that dependency is explicit
and intentional.

Drafting sequence
-----------------
1. Identify what concept this document should own.
2. Define what the document is for.
3. Define what it does and does not define.
4. Define the intended authority boundary.
5. Identify upstream and downstream relationships only when they matter.
6. Write explicit normative prose.
7. Check for over-generalization and scope drift.
8. Check whether a neighboring document would now need a consistency pass.

Required boundary questions
---------------------------
Before drafting, ask:
- Why does this concept need its own document?
- Which document family does it belong to?
- What neighboring document should remain the authority for adjacent meaning?
- What should this new document refuse to define?
- Does this document introduce a new concept or merely restate an existing
  one better handled elsewhere?

Authority-home rules
--------------------
A new document should:
- own one coherent semantic area
- avoid silently taking over neighboring meanings
- avoid duplicating authoritative semantics from another indexed document
- prefer one clear authority home plus references over duplicated rules

Anti-generalization reminder
----------------------------
Do not broaden the document beyond the designed scope merely because a broader
wording sounds cleaner or more reusable.
Write for the design that exists.
If broader future applicability is uncertain, keep the document narrower and
leave extension for later deliberate design.

Drafting quality checks
-----------------------
After drafting, ask:
- Is the scope clear?
- Is the authority boundary clear?
- Is the document defining too much or too little?
- Is any important concept only implied instead of stated?
- Did wording broaden the design beyond what was actually reasoned about?
- Would a future reader know which neighboring document still owns adjacent
  meaning?

End
---
This helper defines reusable drafting workflow only.
It does not define Turnix runtime semantics.
