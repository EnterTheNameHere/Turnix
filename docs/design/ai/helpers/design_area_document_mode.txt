Turnix design-area document mode helper
=======================================

Purpose
-------
This helper defines practical behavior for working in design-area style of
document mode.

Use this helper when:
- working on DA documents primarily
- discussing design semantics rather than code implementation
- integrating locked answers into existing design-area documents
- drafting a new design-area document

Mode distinction
----------------
Design-area document mode is not code mode.
Optimize for:
- semantic clarity
- interpretation stability
- authority boundaries
- cross-document coherence

Do not default to implementation detail unless the design explicitly needs it.

Two main sub-modes
------------------
1. Existing-document integration mode
2. New-document drafting mode

Existing-document integration mode
----------------------------------
When a DA already exists:
- read the current document first
- identify preserved meaning
- identify changed meaning
- identify new or extended meaning
- integrate the new locked meaning without losing valid prior semantics
- perform a drift pass after rewriting
- perform a consistency sweep if several related docs changed

New-document drafting mode
--------------------------
When the DA is new:
- define what the area is for
- define what it does and does not define
- define authority boundaries
- define upstream/downstream relations only when meaningful
- keep scope narrow enough to stay authoritative and stable

DA-specific review questions
----------------------------
After drafting or integrating, ask:
- Did any valid prior rule disappear?
- Did any unchanged meaning become weaker?
- Did the document's center of gravity shift?
- Did authority ownership move to or from another DA?
- Did a term broaden beyond the designed scope?
- Did this DA start duplicating a neighboring DA instead of referencing it?
- Is the relationship between this DA and neighboring DAs still clear?

Working reminder
----------------
A design-area document should be explicit enough that a future reader knows:
- what this document owns
- what this document does not own
- which neighboring document still owns adjacent meaning

End
---
This helper defines reusable DA-mode workflow only.
It does not define Turnix runtime semantics.
