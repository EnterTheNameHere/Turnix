Turnix existing document integration helper
===========================================

Purpose
-------
This helper defines how to integrate new or newly locked meaning into an
already existing Turnix document without losing valid prior semantics.

Use this helper when:
- editing an existing DA, DD, ODA, or IS document
- integrating answers from prior question-hardening work
- replacing only parts of a document while preserving the rest
- updating a document after related docs already changed

Core principle
--------------
Integrate into the existing semantic structure.
Do not rewrite as if the previous document were blank unless a full semantic
replacement is explicitly intended.

Required identification pass
----------------------------
Before rewriting, identify:
- preserved meaning
- changed meaning
- new meaning
- extended meaning
- valid prior meaning that must remain
- text that is currently correct but may be softened by a broader rewrite

Lock-before-write reminder
--------------------------
Do not integrate unstable meaning.
If core questions are still unresolved, return to design discussion and lock
those decisions first.

Integration sequence
--------------------
1. Read the current document.
2. Identify the current authority home and document scope.
3. Mark preserved meaning that should survive unchanged.
4. Mark obsolete, replaced, or incomplete parts.
5. Insert the newly locked meaning into the correct authority location.
6. Check whether any neighboring sections must be tightened to preserve the
   same interpretation.
7. Perform a drift pass.
8. If several related documents changed, perform a consistency sweep.

Preservation rules
------------------
When integrating into an existing document:
- preserve valid prior meaning unless intentionally changing it
- do not erase a correct rule merely because a rewrite is cleaner
- do not broaden a term unless that broader scope was intentionally designed
- do not silently move source-of-truth ownership to another document

Warning signs
-------------
Stop and re-check if the integration:
- changes the document's center of gravity
- changes which document owns a concept
- makes a sharp rule more abstract
- introduces broader terms than the design actually covered
- collapses several separate concepts into one generic phrase
- leaves formerly explicit behavior merely implied

Post-integration drift pass
---------------------------
After integrating, ask:
- Did any valid prior rule disappear?
- Did any unchanged meaning become weaker?
- Did any guarantee become less direct?
- Did source-of-truth ownership move?
- Did any new cross-document dependency appear?
- Did any wording broaden the designed scope?

Output expectations
-------------------
When reporting the result of integration work, distinguish:
- preserved meaning that remains intact
- intentionally changed meaning
- newly introduced meaning
- risks or drift found during the pass

End
---
This helper defines reusable integration workflow only.
It does not define Turnix runtime semantics.
