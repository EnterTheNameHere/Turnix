Turnix question hardening workflow helper
=========================================

Purpose
-------
This helper defines a reusable workflow for turning fuzzy design discussion
into locked, documentable meaning.

Use this helper when:
- design discussion is still ambiguous
- several possible meanings are mixed together
- rewriting would be premature because the semantics are not stable yet
- the next useful step is clarification rather than prose production

Core principle
--------------
Do not let document prose decide design.
Design first, document second.

Hardening workflow
------------------
1. Identify the fuzzy area.
2. Separate it into explicit questions.
3. Resolve one question at a time.
4. Lock each accepted answer.
5. Move to the next question.
6. Only after enough questions are locked, integrate the meaning into
   documents.

Useful question forms
---------------------
Questions are strongest when they ask things such as:
- What exactly is the authority home?
- Is this hard, soft, advisory, or undefined?
- What does the absence of this field or rule mean?
- Is this deterministic behavior or only Pack Manager/UI help?
- Does this apply to all cases or only a narrower class?
- What concrete form are we actually designing for?
- Is this new meaning, changed meaning, or preserved meaning?
- Which neighboring document should own adjacent concepts?

Do not rewrite yet if:
---------------------
- the source of truth is still unclear
- several meanings are still mixed together
- scope is still moving
- broader applicability has not been designed
- a key term is still undecided
- the answer would likely change after one more focused question

Output expectation
------------------
When enough questions are resolved, summarize the locked meaning in a form
usable for later integration or drafting.

End
---
This helper defines reusable design-hardening workflow only.
It does not define Turnix runtime semantics.
