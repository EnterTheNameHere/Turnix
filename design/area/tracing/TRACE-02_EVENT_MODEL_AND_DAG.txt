DESIGN AREA TRACE-02 - TRACE EVENT MODEL AND DAG
===============================================================================

Status:

EVOLVING

Description:
------------
Defines trace event structure, trace DAG semantics, and parent-link meaning.

Purpose:
--------
This document defines the event and causal graph mechanics formerly contained
in DA-03 sections 04 through 06.

It defines:
- trace event structure
- trace DAG semantics
- parent links
- the distinction between causal order and temporal observation

It does NOT define:
- span mechanics
- reason catalogs
- attribute requirements for domain-specific events
- concrete serialization
- trace storage formats

===============================================================================
01. TRACE EVENT MODEL
===============================================================================

01.1. A trace event is one structured evidence record.

01.2. Each trace event has:
      - eventId
      - timestamp
      - level
      - domain
      - message
      - reason
      - attributes
      - optional parentEventIds

01.3. eventId identifies one trace event.

01.4. timestamp records observed emission time.

01.5. timestamp is observational.

01.6. timestamp does not define causality.

01.7. level classifies presentation severity.

01.8. domain classifies the subsystem or subject area that emitted the
      event.

01.9. message provides a human-readable explanation.

01.10. reason provides a machine-readable reason identifier.

01.11. attributes provide structured data required to interpret the event.

01.12. parentEventIds records explicit causal parents when such parents
       exist.

01.13. The exact concrete serialization of trace events is defined outside
       this document.

01.14. Concrete serialization does not change trace event semantics.


===============================================================================
02. TRACE DAG
===============================================================================

02.1. Events form a trace DAG rather than a flat stream.

02.2. "Trace Directed Acyclic Graph" is the canonical name for the causal
      trace structure defined by this document.

02.3. "trace DAG" is the permitted abbreviation.

02.4. The trace DAG is the directed acyclic graph formed by trace events
      and their explicit parent and span-causality links.

02.5. Causal ordering is authoritative within retained trace evidence.

02.6. Causal ordering is represented by explicit DAG links.

02.7. Temporal ordering is observational only.

02.8. Temporal ordering does not determine causality or correctness.

02.9. Turnix does not infer causality from timestamps.

02.10. Cycles do not occur in the trace DAG.

02.11. No trace event is causally reachable from itself through parent or
       span-causality links.

02.12. The trace DAG is not a substitute for authoritative committed state.


===============================================================================
03. PARENT LINKS
===============================================================================

03.1. A trace event may reference zero, one, or multiple parent events.

03.2. parentEventIds, when present, is the set of explicit causal parents
      of that trace event.

03.3. Multiple parent links are permitted.

03.4. Absence of parent links means independent trace origin.

03.5. Absence of parent links does not imply simultaneity.

03.6. A trace event does not list the same parent event more than once.

03.7. Parent links express causal structure rather than temporal order.


===============================================================================
04. LOCKED SUMMARY
===============================================================================

04.1. LOCKED:
      - a trace event is one structured evidence record
      - event timestamp is observational
      - timestamp does not define causality
      - the trace DAG is the canonical causal structure
      - temporal order does not define causality
      - parent links express explicit causal parentage
      - the trace DAG is not committed state
