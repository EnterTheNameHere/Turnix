===============================================================================
01. TRACE SPANS
===============================================================================

01.1. Trace spans represent bounded work with a start and an end.

01.2. A span has a begin event and an end event.

01.3. Span begin and span end events are trace events.

01.4. Span begin and span end events participate in the trace DAG.

01.5. Span begin occurs before span end in the same trace DAG.

01.6. A span end does not exist without a reachable span begin.

01.7. Span begin and span end share a stable span identity attribute.

01.8. The exact span identity encoding remains OPEN.

01.9. Span begin and span end are connected by explicit causality links.

01.10. Any export, snapshot, or offline trace view that includes span end
       also includes the corresponding span begin.

01.11. If a snapshot includes trace evidence, it satisfies the rule:
       "no included span end without included span begin."

01.12. This snapshot requirement is [satisfied-by: DA-08 11.9].


===============================================================================
02. TRACE ROOTS AND SPAN STRATIFICATION
===============================================================================

02.1. Trace storage may contain multiple unconnected trace DAG roots.

02.2. Each root represents an independent trace origin.

02.3. Unconnected roots are expected and remain representable without
      ambiguity.

02.4. Root spans identify the outer retained causal region for trace
      evidence.

02.5. Nested spans identify bounded work inside a retained causal region.

02.6. Span stratification allows trace consumers to distinguish broad
      runtime regions from nested bounded work.

02.7. A subject-owning document may define subject-specific span kinds.

02.8. Subject-specific span kinds do not redefine the span mechanics in
      this document.

02.9. A subject-specific span remains a trace span when it follows the
      begin, end, identity, and causal-link rules in this document.

02.10. TRACE owns the trace-substrate reason identifiers used for kernel-run
       trace root spans.

02.11. The following kernel-run trace span reasons are defined:

       - KernelRunSpanBegin
       - KernelRunSpanEnd

02.12. KernelRunSpanBegin is emitted when Turnix begins a kernel run.

02.13. KernelRunSpanBegin trace evidence records enough context to explain:
       - kernelRunId
       - spanIdentity

02.14. KernelRunSpanEnd is emitted when Turnix ends a kernel run.

02.15. KernelRunSpanEnd trace evidence records enough context to explain:
       - kernelRunId
       - spanIdentity
       - endReasonCategory

02.16. KernelRunSpanEnd endReasonCategory distinguishes:
       - NormalExit
       - Crash
       - ForcedExit
       - Restart

03. OPEN QUESTIONS
===============================================================================

03.1. OPEN questions include:

      - cross-session trace stitching for root spans

04. LOCKED SUMMARY
===============================================================================

04.1. LOCKED:
      - trace spans have begin and end events
      - span end does not exist without a reachable span begin
      - span begin and span end share a stable span identity attribute
      - exported or snapshotted trace evidence does not include span end
        without the corresponding span begin
      - trace roots and span stratification are representable
      - subject-specific span kinds do not redefine substrate span mechanics
      - TRACE owns kernel-run trace root span reason identifiers
